# 보드미 (Bodeumi) — 아기 사진 앱 프로젝트 기획서

> 작성일: 2026-04-13 | 최종 수정: 2026-04-13

---

## 1. 전체 아키텍처

```
[업로더 (부모)]          [뷰어 (가족/조부모)]
 스마트폰                   스마트폰
     │                         │
     │  카메라 촬영              │  앨범 조회
     │  / 갤러리 선택            │  (뷰어 권한)
     ▼                         ▼
[보드미 Flutter 앱] ←──────────────────────────
     │                         앱 공유 링크 / 초대 코드
     │  HTTPS (Multipart Form / REST API)
     │  ← 업로드 진행률(Progress) 표시
     ▼
[FastAPI 서버]
     │
     ├─ EXIF 날짜 추출
     ├─ 썸네일 생성 (Pillow)
     ├─ 월별 폴더 정리
     ├─ 사용자/멤버 관리 (JWT 인증)
     ├─ 초대 코드 생성 및 검증
     │
     ▼
[파일 저장소]                [DB (SQLite / PostgreSQL)]
  /photos                     ├── users (사용자)
    ├── 2026-01/               ├── albums (앨범)
    │     ├── original/        ├── album_members (멤버/권한)
    │     └── thumbnails/      ├── invitations (초대 코드)
    ├── 2026-02/               └── photos (메타데이터)
    └── ...
```

**흐름 요약**

1. 부모(관리자)가 앱에서 사진 촬영 또는 갤러리 선택 후 업로드
2. 서버가 EXIF 메타데이터에서 촬영 날짜 추출, 월별 폴더에 저장
3. Pillow로 썸네일 자동 생성
4. 부모가 초대 링크 또는 6자리 코드를 가족/조부모에게 공유
5. 초대받은 가족이 앱에서 코드 입력 → 앨범 멤버로 등록
6. 권한에 따라 사진 업로드(업로더) 또는 조회만(뷰어) 가능

## 2. 추천 기술 스택

### 백엔드: FastAPI (Python)

| 항목 | 내용 |
|------|------|
| 언어 | Python 3.11+ |
| 프레임워크 | **FastAPI** |
| 이미지 처리 | **Pillow** (썸네일 생성, EXIF 읽기) |
| 파일 저장 | 로컬 디스크 or 클라우드 스토리지(S3 등) |
| 인증 | **JWT** (Access Token + Refresh Token) |
| 데이터베이스 | **SQLite** (초기) → PostgreSQL (확장 시) |
| ORM | **SQLModel** (FastAPI 제작자가 만든 Python ORM) |
| 서버 실행 | **Uvicorn** (ASGI) |

공유 앨범 기능 추가로 사용자·권한 데이터를 관리해야 하므로 **DB가 필수**입니다. SQLite는 별도 설치 없이 바로 사용할 수 있어 초기 개발에 적합합니다.

## 3. 주요 기능 목록

### 앱 (Flutter)

#### 기본 사진 기능

| 기능 | 설명 |
|------|------|
| **사진 선택** | 갤러리에서 단일/다중 사진 선택 (`image_picker`) |
| **사진 촬영** | 카메라 앱 연동 후 즉시 업로드 |
| **업로드** | HTTP Multipart 요청, 백그라운드 업로드 지원 (`dio`) |
| **업로드 진행률** | LinearProgressIndicator로 실시간 퍼센트 표시 |
| **월별 갤러리 뷰** | 탭 or 드롭다운으로 월 선택, 썸네일 그리드 표시 |
| **사진 상세 보기** | 풀스크린 뷰어, 좌우 스와이프 |
| **오프라인 대응** | 네트워크 없을 때 업로드 대기열 유지 |

#### 공유 앨범 기능

| 기능 | 설명 |
|------|------|
| **초대 코드 생성** | 관리자(부모)가 6자리 코드 또는 딥링크 생성 |
| **링크 공유** | 카카오톡/문자 등으로 초대 링크 공유 (`share_plus`) |
| **코드 입력 화면** | 가족이 코드 입력 → 자동 앨범 참여 |
| **멤버 목록 조회** | 현재 앨범 참여 중인 가족 목록 표시 |
| **권한 표시** | 내 권한(업로더/뷰어) 화면에 표시 |
| **멤버 제거** | 관리자가 특정 멤버를 앨범에서 내보내기 |
| **권한 변경** | 관리자가 멤버의 권한을 업로더 ↔ 뷰어로 변경 |

---

### 서버 (FastAPI)

#### 기본 사진 API

| 기능 | 설명 |
|------|------|
| **사진 수신** | Multipart 파일 수신, JPEG/PNG/HEIC 지원 |
| **EXIF 날짜 추출** | `piexif` or `Pillow`로 촬영 날짜 읽기 (없을 시 업로드 날짜 사용) |
| **월별 폴더 정리** | `YYYY-MM/original/` 경로에 자동 저장 |
| **썸네일 생성** | Pillow로 300×300px 썸네일 자동 생성 |
| **갤러리 API** | 월별 사진 목록 반환 (썸네일 URL 포함) |
| **사진 삭제** | 원본 + 썸네일 동시 삭제 (권한 확인 후) |
| **중복 방지** | 파일 해시(MD5) 기반 중복 업로드 차단 |

#### 공유 앨범 API (신규)

| 메서드 | 엔드포인트 | 설명 |
|--------|-----------|------|
| `POST` | `/auth/register` | 회원가입 (이름, 이메일, 비밀번호) |
| `POST` | `/auth/login` | 로그인 → JWT Access/Refresh Token 반환 |
| `POST` | `/auth/refresh` | Access Token 갱신 |
| `POST` | `/invite/create` | 초대 코드 생성 (만료 시간 설정 가능, 관리자만) |
| `GET` | `/invite/{code}` | 초대 코드 유효성 확인 + 앨범 정보 미리보기 |
| `POST` | `/invite/{code}/accept` | 코드 입력 → 앨범 멤버로 등록 (권한: viewer 기본) |
| `GET` | `/album/members` | 앨범 멤버 목록 조회 (이름, 권한, 가입일) |
| `PATCH` | `/album/members/{user_id}/role` | 멤버 권한 변경 (viewer ↔ uploader, 관리자만) |
| `DELETE` | `/album/members/{user_id}` | 멤버 제거 (관리자만) |
| `GET` | `/album/my-role` | 내 현재 권한 조회 |

#### 권한 체계

```
관리자 (Admin)
  └─ 초대 코드 생성, 멤버 관리, 사진 업로드/삭제 모두 가능

업로더 (Uploader)
  └─ 사진 업로드, 조회 가능 / 멤버 관리 불가

뷰어 (Viewer)
  └─ 사진 조회만 가능 / 업로드·삭제·멤버 관리 불가
```

## 4. 개발 단계

### Phase 1 — 핵심 기능 MVP (4~6주)

**목표**: 사진 찍고 서버에 올리는 것이 작동하는 버전

- [ ] FastAPI 서버 기본 설정 (업로드 엔드포인트)
- [ ] EXIF 날짜 추출 + 월별 폴더 저장 로직
- [ ] Pillow 썸네일 생성
- [ ] Flutter 앱 — 카메라/갤러리 선택 화면
- [ ] Flutter 앱 — 업로드 기능 + 진행률 표시
- [ ] Flutter 앱 — 월별 갤러리 그리드 뷰
- [ ] Flutter 앱 — 사진 상세 보기 / 스와이프
- [ ] 서버 배포 (Oracle Free / 로컬 테스트)

### Phase 2 — 인증 및 공유 앨범 (4~5주)

**목표**: 가족이 함께 앨범을 볼 수 있는 공유 기능 완성

- [ ] DB 모델 설계 (User, Album, AlbumMember, Photo, Invitation)
- [ ] 서버 — JWT 인증 (회원가입/로그인/토큰 갱신)
- [ ] 서버 — 초대 코드 생성 API (만료 시간 포함)
- [ ] 서버 — 초대 코드 수락 → 멤버 등록 API
- [ ] 서버 — 멤버 권한 관리 API (조회/변경/삭제)
- [ ] 서버 — 권한별 API 접근 제어 미들웨어
- [ ] Flutter 앱 — 로그인/회원가입 화면
- [ ] Flutter 앱 — 초대 코드 입력/공유 화면
- [ ] Flutter 앱 — 멤버 관리 화면 (관리자용)

### Phase 3 — 안정화 및 부가 기능 (2~3주)

**목표**: 실제 사용에 견딜 수 있게 다듬기

- [ ] 오프라인 업로드 대기열
- [ ] 중복 사진 방지 (파일 해시 비교)
- [ ] 사진 삭제 기능 (권한 확인)
- [ ] 초대 코드 만료 처리 및 재발급
- [ ] 에러 처리 및 사용자 알림 개선
- [ ] 푸시 알림 — 새 사진 업로드 시 가족에게 알림 (선택사항)
- [ ] 백업/내보내기 기능 (선택사항)
